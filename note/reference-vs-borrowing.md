# Rust 引用与借用：`&T` 与 Borrowing

> `&s` 是引用（语法），把 `&s` 传给别人的行为叫借用（语义）。两者是同一件事的两个视角。

---

## 一、核心定义

```
引用（Reference） —— &s 这个表达式创造了一个引用，名词，类型是 &T
借用（Borrowing） —— 对 &s 的使用就叫借用，动词，发生在具体场景中
```

```rust
let s = String::from("hello");

let r = &s;        // &s 创造了引用
let r2 = r;        // 使用 r（赋值）—— 借用
use_it(&s);        // 使用 &s（传入函数）—— 借用
let len = r.len(); // 使用 r（方法调用/计算）—— 借用
println!("{}", r); // 使用 r（格式化输出）—— 借用
```

**一句话总结**：`&s` 创造引用，对 `&s` 的使用（传入函数、赋值、计算等）就叫借用。

---

## 二、一句话区分

```
引用（Reference） —— &T 这个类型，名词
借用（Borrowing） —— 用 &s 把数据临时给别人，动词
```

```rust
let s = String::from("hello");

let r: &String = &s;   // &s 创建了引用，同时完成了借用
use_it(&s);            // 把 s 借给 use_it 函数
```

---

## 二、为什么需要"借用"这个概念

其他语言也有引用（C++ `&`），但不叫借用。因为 Rust 的 `&` 绑定了**所有权规则**——不只是"不复制"，而是"**你不拥有它，用完必须还**"。

```rust
fn main() {
    let s = String::from("hello");

    // 按值传递 —— 所有权转移，s 没了
    takes_ownership(s);
    // println!("{}", s);  // ❌ 编译错误

    let s2 = String::from("world");
    // 借用传递 —— 数据还是我的
    borrows(&s2);
    println!("{}", s2);   // ✅ s2 还在
}

fn takes_ownership(s: String) {}   // 参数是 String，我要拥有它
fn borrows(s: &String) {}          // 参数是 &String，我只是借用
```

## 三、借用的两条铁律

```rust
let mut data = String::from("hello");

// 规则 1：同一时刻，只能有 1 个 &mut T 或任意多个 &T
let r1 = &data;
let r2 = &data;        // ✅ 多个不可变引用 OK

// let m1 = &mut data; // ❌ 不可变引用存活时不能有可变引用

// 规则 2：引用必须始终有效（不能悬垂）
// let r = dangle();   // ❌ 编译错误

fn dangle() -> &String {
    let s = String::from("bye");
    &s  // s 函数结束被 drop，返回的引用指向已释放的内存
}
```

## 四、借用在哪些地方发生

**只要写了 `&`，借用就发生了**——不只是函数参数。

```rust
let s = String::from("hello");

let r = &s;             // 绑定到变量 —— 借
println!("{}", &s);     // 传给宏         —— 借
some_fn(&s);            // 传给函数参数   —— 借
vec.push(&s);           // 传给方法       —— 借
```

---

## 五、可变借用

```rust
let mut s = String::from("hello");

change(&mut s);
println!("{}", s);  // 输出：hello, world

fn change(s: &mut String) {
    s.push_str(", world");  // 可以修改，但不拥有
}
```

可变借用的额外约束：**借用期间，原拥有者不能访问数据**。

```rust
let mut s = String::from("hello");
let r = &mut s;

// println!("{}", s);  // ❌ s 已被可变借用，暂时不能访问
r.push_str(", world");
println!("{}", r);    // ✅ 通过可变引用访问
// 借用结束，s 恢复可用
println!("{}", s);    // ✅
```

## 六、借用的生命周期

```
    main                     some_fn
     │                         │
     │── &s ─────────────────→│  借出
     │                        │  使用数据
     │←───────────────────────│  函数返回 = 隐式归还
     │                         │
   s 可用                   参数消失（引用本身销毁）
```

借用是**临时的**——借方（函数/变量）离开作用域时，借用关系自动解除，数据回到拥有者手中。

## 七、实战：函数签名怎么设计

```rust
// ❌ 只读却要 &String —— 太窄了
fn bad(s: &String) { println!("{}", s); }

// ✅ 只读用 &str —— 通吃所有字符串来源
fn good(s: &str) { println!("{}", s); }

// ✅ 需要修改用 &mut String
fn mutate(s: &mut String) { s.push('!'); }

// ✅ 需要拥有数据用 String
fn own(s: String) -> String { s }
```

## 八、双层引用：`let a = "hola"; let b = &a;`

`"hola"` 是字符串字面量，类型本身就是 `&'static str`。所以：

```rust
let a = "hola";   // a: &str
let b = &a;       // b: &&str
```

内存布局：

```
  只读数据段 (.rodata)              栈
┌─────────────────┐           ┌─────────────────┐
│ h o l a \0      │           │ a: &str (16B)   │
└─────────────────┘           │ ptr ────────────→ ─→ "hola" 的地址
                              │ len: 4          │
                              └────────┬────────┘
                                       │
                              ┌────────┴────────┐
                              │ b: &&str (8B)   │
                              │ ptr ────────────→ ─→ a 的地址
                              └─────────────────┘
```

| 变量 | 类型 | 大小 | 存了什么 |
|------|------|------|----------|
| `a` | `&str` | 16 字节 | `(ptr → .rodata, len = 4)` |
| `b` | `&&str` | 8 字节 | `(ptr → a)` |

`a` 是胖指针，直接指向只读段的 `"hola"`。`b` 是瘦指针，指向栈上的 `a`。要通过 `b` 拿到字符串，需要**两次解引用**：`b` → `a` → `"hola"`。

**关键点**：

- `"hola"` 没有创建任何拥有者，数据在编译期就嵌进二进制的只读段，生命周期是 `'static`
- `&&str` 几乎没有实用价值——多了一层间接，没有实际收益

---

"`★ Insight ───────────────────────────────────────────`
1. **`&s` 出现的地方就是借用发生的地方**，不管是在函数调用、变量绑定还是宏展开中
2. **借用检查器不是检查类型，是检查关系**——"A 拥有数据，B 借用了它，B 不能比 A 活得更久，可变借用期间别人不能读"
3. **函数参数默认 move（转移所有权），`&` 是例外**——这正是"借用"这个词存在的意义：其他语言传参要么拷贝要么转移，Rust 多了一种"借给你用用就还"
4. **引用本身也是值，可以再取引用**。`&&T` 就是指向 `&T` 的指针，8 字节瘦指针。每次多一层 `&` 就多一次解引用
