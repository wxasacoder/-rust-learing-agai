# Rust 杂记：所有权、借用规则、栈堆分配

> 零散但核心的概念汇总。

---

## 一、没有 GC，也不需要手动回收

Rust 不像 Java/Go 有运行时垃圾回收，也不需要像 C 那样手动 `malloc`/`free`。它依靠的是**所有权机制**在编译期管理内存。

```
编译期（所有权检查）  →  运行期（自动 drop）
       ↓                      ↓
   不需要 GC              不需要手动 free
```

---

## 二、所有者对数据负责

**每个数据都有一个所有者（owner），所有者负责在作用域结束时释放（drop）数据。**

```rust
{
    let s = String::from("hello");   // s 是所有者
    // ... 使用 s
}                                     // s 离开作用域，drop "hello" 的堆内存
```

---

## 三、一个数据只能有一个所有者

```rust
let s1 = String::from("hello");
let s2 = s1;        // 所有权从 s1 转移到 s2

// println!("{}", s1);  // ❌ s1 已失效
println!("{}", s2);     // ✅ s2 是现在的所有者
```

这就是 **Move 语义**。

---

## 四、只读引用可有多个，可变引用只能有一个

```rust
let s = String::from("hello");

let r1 = &s;       // 不可变引用
let r2 = &s;       // 可以同时存在多个
println!("{} {}", r1, r2);

let mut s2 = String::from("hello");
let m1 = &mut s2;  // 可变引用
*m1 += " world";
// let m2 = &mut s2;  // ❌ 不能同时存在多个可变引用
```

这就是著名的**借用规则**：

| 引用类型 | 同时存在的数量 |
|---|---|
| `&T`（不可变引用） | 多个 |
| `&mut T`（可变引用） | 一个 |
| `&T` + `&mut T` 混合 | 不能共存 |

---

## 五、传入函数或赋值，所有权发生转移

```rust
fn take_ownership(s: String) {
    // s 现在是函数的所有者
} // 函数结束，s 被 drop

let s = String::from("hello");
take_ownership(s);   // 所有权从 s 转移到函数参数
// println!("{}", s);  // ❌ s 已失效
```

```rust
let s1 = String::from("hello");
let s2 = s1;         // 所有权从 s1 转移到 s2
```

**赋值和传参的本质都是 Move**：把值的所有权从一个变量转移到另一个变量。

## 六、完整内存模型图

### 场景：创建 String，创建引用，传入函数

```rust
let s = String::from("hello");
let r1 = &s;
let r2 = &s;
print_refs(r1, r2);
// s 仍在这里
```

```
  栈帧（main 函数）                              堆
┌─────────────────────────┐              ┌──────────────────┐
│ s (String, 24字节):      │              │ h e l l o \0     │
│   ptr ──────────────────────→            └──────────────────┘
│   len: 5                 │
│   cap: 5                 │
│                          │
│ r1: &str (16字节):       │
│   ptr ──────────────┐    │
│   len: 5            │    │
│                     │    │
│ r2: &str (16字节):       │
│   ptr ──────────────┘    │
│   len: 5                 │
└─────────────────────────┘

关键：
  s 是所有者 → 负责 drop 堆数据
  r1, r2 是借用者 → 只读，不负责释放
  r1.ptr 和 r2.ptr 指向同一个地址
```

### 传入函数后

```rust
fn take_ownership(s: String) {
    // s 在此处是所有者
}
```

```
  调用前（main 的栈帧）            调用后（take_ownership 的栈帧）
┌─────────────────────┐         ┌─────────────────────┐
│ s (旧所有者)         │         │ s (新所有者)         │
│   已失效！            │   →     │   ptr ───────→ 堆数据  │
│   ptr: (无效)        │         │   len: 5           │
│   len: (无效)        │         │   cap: 5           │
└─────────────────────┘         └─────────────────────┘
                                        ↓
                                   堆: hello
                                   函数结束时 drop

关键：
  所有权从 main 的 s 转移到函数的 s
  同一个堆数据，只能有一个所有者
  函数结束，所有者消失，堆数据被释放
```

### 可变引用独占性

```rust
let mut s = String::from("hello");
let r1 = &s;        // 不可变借用
// let m1 = &mut s;  // ❌ 此时不能有可变借用
drop(r1);           // r1 生命周期结束
let m1 = &mut s;    // ✅ 现在可以可变借用了
*m1 += " world";
```

```
  栈帧                                         堆
┌─────────────────────────┐              ┌──────────────────┐
│ s (String, 所有者)       │              │ h e l l o \0     │
│   ptr ──────────────────────→            └──────────────────┘
│   len: 5                 │
│   cap: 5                 │
│                          │
│ m1: &mut String (8字节): │
│   ptr ──────────────────→ (指向 s 本身)
│                          │
│  独占借用期间，s 不可访问  │
└─────────────────────────┘

关键：
  可变引用 &mut T 独占数据的访问权
  借用期间，所有者不能读也不能写
  借用结束，所有者恢复完整访问权
```

## 七、逐个变量的内存模型

下面这段代码中每个变量的内存长什么样？

```rust
let a = 1;
let b = "hello";
let c = String::from("hello");
let d = &a;
let e: &str = "hello";
let f = &e;
```

### 1. `let a = 1;`

```rust
let a = 1;     // 类型推断为 i32
```

```
  栈
┌─────────┐
│ a: 1    │   4 字节，值直接存在栈上
└─────────┘

关键：
  类型: i32（Sized，编译期确定大小）
  分配: 栈上
  所有者: a
  没有指针，数据就是值本身
```

---

### 2. `let b = "hello";`

```rust
let b = "hello";     // 类型推断为 &str
```

```
  栈                          .rodata（二进制只读数据段）
┌───────────────┐            ┌──────────────────┐
│ b: &str       │            │ 68 65 6c 6c 6f    │
│   ptr ────────────→          │ h  e  l  l  o    │
│   len: 5      │            └──────────────────┘
└───────────────┘
     16 字节

关键：
  类型: &str（胖指针，16 字节）
  数据: 编译期嵌入可执行文件的 .rodata 段
  所有者: b 是这个胖指针的所有者，但不拥有字符串数据
  数据在程序启动时就存在，不会被 drop
```

---

### 3. `let c = String::from("hello");`

```rust
let c = String::from("hello");     // 类型: String
```

```
  栈                                  堆
┌─────────────────────┐          ┌──────────────────┐
│ c: String (24字节)   │          │ h e l l o \0     │
│   ptr ──────────────────→        └──────────────────┘
│   len: 5             │
│   cap: 5             │
└─────────────────────┘

关键：
  类型: String（结构体，24 字节）
  数据: 运行时在堆上分配
  所有者: c 拥有堆上的 "hello"
  c 离开作用域时，堆数据被 drop
```

---

### 4. `let d = &a;`

```rust
let a = 1;
let d = &a;     // 类型: &i32
```

```
  栈（假设 a 在 0x7fff1000，d 在 0x7fff1004）
┌─────────────────────────┐
│ a (0x7fff1000):  1      │   ← 值直接存在栈上，4 字节
│                         │
│ d (0x7fff1004):         │   ← d 存储的就是 a 的地址
│   值: 0x7fff1000 ───────┘   （8 字节瘦指针）
└─────────────────────────┘

关键：
  类型: &i32（瘦指针，8 字节）
  d 的值 = a 的内存地址
  读 d 就是读 a 的地址，*d = 1
  i32 实现了 Copy，&a 是取地址，不是 Move
  d 不能比 a 活得更久（生命周期检查）
```

---

### 5. `let e: &str = "hello";`

```rust
let e: &str = "hello";     // 类型: &str（和 b 完全一样）
```

```
  栈                          .rodata（二进制只读数据段）
┌───────────────┐            ┌──────────────────┐
│ e: &str       │            │ 68 65 6c 6c 6f    │
│   ptr ────────────→          │ h  e  l  l  o    │
│   len: 5      │            └──────────────────┘
└───────────────┘
     16 字节

关键：
  和 b 的内存布局完全一样
  区别只是 b 是类型推断，e 是显式标注类型
  数据在 .rodata，指针在栈上
```

---

### 6. `let f = &e;`

```rust
let e: &str = "hello";
let f = &e;     // 类型: &&str（引用的引用）
```

```
  栈
┌───────────────────┐
│ f: &&str (8字节)   │
│   ptr ────────→ ┌───────────────┐
└───────────────────┘             │ e: &str (16字节)  │        .rodata
                                  │   ptr ───────→      │      ┌──────────────┐
                                  │   len: 5            │      │  h e l l o   │
                                  └───────────────┘      └──────┘

关键：
  类型: &&str（瘦指针，8 字节）
  f 是指向 e 的指针，e 本身是一个胖指针
  所以需要两次解引用才能到达 "hello" 的数据
  f 的栈大小只有 8 字节，因为它只指向 e（已知大小的 &str）
  &&str 不是胖指针！因为 &str 的大小是已知的（16 字节）
```

---

### 总览对比

```
变量    类型      栈上大小    数据在哪          所有者？
─────────────────────────────────────────────────────
a       i32       4 字节      栈上（值本身）     a 拥有值
b       &str      16 字节     .rodata           b 拥有胖指针，不拥有数据
c       String    24 字节     堆                c 拥有堆数据
d       &i32      8 字节      栈上（a 的位置）   d 拥有引用，借用自 a
e       &str      16 字节     .rodata           同 b
f       &&str     8 字节      栈上（e 的位置）   f 拥有引用，借用自 e
```

## 八、切片的本质：造胖指针

> 切片本身会创造一个胖指针：指针指向真实数据的起始地址，额外携带的是**内存地址的范围**（起始 + 长度 = 终点），合起来就是对真实数据的一个切片视图。

```rust
let arr = [10, 20, 30, 40, 50];
let slice = &arr[1..4];     // 切片操作 → 造胖指针
```

```
  slice（栈，16 字节胖指针）
┌─────────────────────────┐
│ ptr: 0x7fff2004 ────────┐   ← 指向真实数据的起始地址
│ len: 3                  │   ← 从起始地址开始的范围
└─────────────────────────┘   │
                              │
  arr（栈）                    │
┌─────────────────────────┐   │
│ [0] 10  (0x7fff2000)    │   │
│ [1] 20  (0x7fff2004) ───┘   ← slice.ptr 指向这里
│ [2] 30  (0x7fff2008)         │
│ [3] 40  (0x7fff200c)         │
│ [4] 50  (0x7fff2010)         │
└─────────────────────────┘    ↑
                               slice.len = 3，覆盖 3 个 i32
                               地址范围: 0x7fff2004 ~ 0x7fff2010

关键：
  切片操作不复制数据，只是创造新的 (ptr, len)
  ptr = 数据的起始地址
  len = 从这个地址开始往后数多少个元素
  ptr + len = 切片的结束地址
  这就是"切片"的全部含义：一段连续内存的地址范围
```

### 字符串切片同理

```rust
let s = String::from("hello");
let slice: &str = &s[1..4];    // "ell"
```

```
  slice（栈，16 字节胖指针）
┌─────────────────────────┐
│ ptr: s.ptr + 1 ────────→   ← 从 'e' 开始
│ len: 3                   │   ← 3 个字节
└─────────────────────────┘
            ↓
  堆（String 拥有的数据）
┌─────────────────────────┐
│ h │ e │ l │ l │ o │ \0  │
    ↑           ↑
    ptr+1     ptr+4 (结束)
    
地址范围: (s.ptr + 1) ~ (s.ptr + 4)
```

### 一句话总结

**切片 = (起点地址, 长度) = 一段连续内存的地址范围 = 对原始数据的零拷贝视图。**
